Module Picos_threaded

A basic Thread based implementation of the effects based operations of Picos.

Briefly:

  • Fiber.current uses TLS to store the current fiber.
  • Fiber.spawn creates a new Thread for each fiber.
  • Fiber.yield just calls Thread.yield.
  • Computation.cancel_after uses DLS to store a priority queue of timeouts and a per-domain background timeout thread that runs a Unix.select loop to cancel computations.
  • Trigger.await uses TLS to store a Mutex and Condition to suspend the thread.

The operations initialize their resources, per-thread, and per-domain state and background timeout thread, only when actually used. If the default cancel_after is not used, no background timeout thread will be created and no per-domain state will be used. If none of the defaults are used, no per-thread state will be used.
