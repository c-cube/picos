Module Picos_threaded

A basic Thread based implementation of the effects based operations of Picos.

Briefly:

  • current returns the current fiber from the per thread state.
  • spawn creates a new Thread for each fiber.
  • yield just calls Thread.yield.
  • cancel_after uses DLS to store a priority queue of timeouts and a per-domain background timeout thread that runs a Unix.select loop to cancel computations.
  • await uses a per thread mutex and condition to suspend the thread.
